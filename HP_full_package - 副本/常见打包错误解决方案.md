# XRD 应用打包 - 常见错误解决方案

## 🔴 错误 1: fabio.pilatusimage 模块找不到

### 错误信息
```
ModuleNotFoundError: No module named 'fabio.pilatusimage'
```

或类似的错误：
```
ModuleNotFoundError: No module named 'fabio.edfimage'
ModuleNotFoundError: No module named 'fabio.tifimage'
```

### 问题原因

**这是 fabio 包的动态导入问题！**

fabio 是一个用于读取各种 XRD 检测器图像格式的库。它支持 30+ 种图像格式（Pilatus、EDF、TIFF、MarCCD、CBF 等）。

问题在于：
1. **fabio 使用动态导入**：在运行时根据文件扩展名动态加载对应的格式处理模块
2. **PyInstaller 无法自动检测**：静态分析无法发现这些动态导入的模块
3. **结果**：打包后的 EXE 文件缺少这些格式模块，运行时报错

### ✅ 解决方案（已修复）

我们通过以下方式解决了这个问题：

#### 1. 更新 xrd_app.spec - 添加所有 fabio 格式模块

在 `hiddenimports` 列表中明确添加了所有 fabio 图像格式：

```python
hiddenimports = [
    # ... 其他模块 ...

    # fabio - 所有图像格式模块（关键！）
    'fabio.pilatusimage',      # Pilatus detector
    'fabio.edfimage',          # EDF format
    'fabio.tifimage',          # TIFF format
    'fabio.marccdimage',       # MarCCD detector
    'fabio.cbfimage',          # CBF format
    'fabio.brukerimage',       # Bruker format
    'fabio.mar345image',       # MAR345 detector
    'fabio.hdf5image',         # HDF5 format
    'fabio.eigerimage',        # Dectris Eiger
    # ... 以及其他 20+ 种格式 ...
    'fabio.fabioformats',      # 格式注册器（非常重要！）
]
```

#### 2. 创建 PyInstaller Hook 文件

创建了 `hooks/hook-fabio.py`，自动收集所有 fabio 子模块：

```python
from PyInstaller.utils.hooks import collect_submodules

# 自动收集所有 fabio 子模块
hiddenimports = collect_submodules('fabio')
```

#### 3. 同样处理 pyFAI

pyFAI 也有类似的动态导入问题，创建了 `hooks/hook-pyFAI.py`。

### 📂 项目结构

确保您的项目包含以下文件：

```
HP_full_package - 副本/
├── xrd_app.spec           ✅ 已更新（包含所有隐藏导入）
├── hooks/                 ✅ 新增目录
│   ├── hook-fabio.py     ✅ fabio 的 PyInstaller hook
│   └── hook-pyFAI.py     ✅ pyFAI 的 PyInstaller hook
├── build.bat
├── build.sh
└── ... 其他文件 ...
```

### 🚀 重新打包

使用更新后的配置重新打包：

```bash
# Windows
build.bat

# Linux/Mac
bash build.sh
```

'pyFAI.azimuthalIntegrator',

## 🔴 错误 3: DLL 加载失败（Windows）

### 错误信息
```
ImportError: DLL load failed while importing _multiarray_umath
OSError: [WinError 126] 找不到指定的模块
```

### 问题原因

某些科学计算库（NumPy、SciPy）依赖 Visual C++ 运行库。

### ✅ 解决方案

**方法 1**：安装 Visual C++ Redistributable
- 下载：https://aka.ms/vs/17/release/vc_redist.x64.exe
- 安装后重试

**方法 2**：使用 Anaconda/Miniconda 环境打包
- Anaconda 自带所有必需的运行库

**方法 3**：修改 spec 文件包含 DLL
```python
import sys
from PyInstaller.utils.hooks import collect_dynamic_libs

binaries = collect_dynamic_libs('numpy')
binaries += collect_dynamic_libs('scipy')
```

## 🔴 错误 4: 图标文件找不到

### 错误信息
```
FileNotFoundError: [Errno 2] No such file or directory: 'D:\\HEPS\\...'
```

### 问题原因

`main.py` 中硬编码了绝对路径：

```python
icon_path = r'D:\HEPS\ID31\dioptas_data\github_felicity\batch\\HP_full_package\ChatGPT Image.ico'
```

### ✅ 解决方案

修改 `main.py`，使用相对路径和 PyInstaller 兼容的资源定位：

```python
import os
import sys

def get_resource_path(relative_path):
    """获取资源文件的绝对路径（支持开发环境和打包后）"""
    try:
        # PyInstaller 创建的临时文件夹
        base_path = sys._MEIPASS
    except AttributeError:
        # 开发环境
        base_path = os.path.abspath(".")
    return os.path.join(base_path, relative_path)

# 使用方法
icon_path = get_resource_path('ChatGPT Image.ico')
if os.path.exists(icon_path):
    root.iconbitmap(icon_path)
```

## 🔴 错误 5: tkinter 找不到

### 错误信息
```
ModuleNotFoundError: No module named '_tkinter'
```

### 问题原因

- **Linux**: 系统未安装 python3-tk
- **Windows**: Python 安装时未勾选 tcl/tk

### ✅ 解决方案

**Linux (Ubuntu/Debian)**:
```bash
sudo apt-get install python3-tk
```

**macOS**:
```bash
brew install python-tk
```

**Windows**:
- 重新运行 Python 安装器
- 选择 "Modify"
- 勾选 "tcl/tk and IDLE"

## 🔴 错误 6: OpenCV 相关错误

### 错误信息
```
ImportError: DLL load failed: 找不到指定的模块 (cv2)
```

### 问题原因

opencv-python 包较大且依赖复杂。

### ✅ 解决方案

**方法 1**：如果不需要 OpenCV，在 spec 中排除
```python
excludes=['cv2', 'opencv']
```

**方法 2**：使用 opencv-python-headless
```bash
pip uninstall opencv-python
pip install opencv-python-headless
```

**方法 3**：添加 OpenCV DLL
```python
from PyInstaller.utils.hooks import collect_dynamic_libs
binaries = collect_dynamic_libs('cv2')
```

## 🔴 错误 7: HDF5 文件读取错误

### 错误信息
```
OSError: Unable to open file (unable to open file: name = '...', errno = 2)
```

### 问题原因

h5py 依赖 HDF5 C 库。

### ✅ 解决方案

确保在 spec 中包含：
```python
hiddenimports=[
    'h5py',
    'h5py.defs',
    'h5py.utils',
    'h5py.h5ac',
    'h5py._proxy',
]

binaries = collect_dynamic_libs('h5py')
```

## 🔴 错误 8: Matplotlib 后端错误

### 错误信息
```
ImportError: Failed to import any of the following matplotlib backends: TkAgg, Agg
```

### ✅ 解决方案

在 spec 中明确包含 matplotlib 后端：

```python
hiddenimports=[
    'matplotlib.backends.backend_tkagg',
    'matplotlib.backends.backend_agg',
    'matplotlib.figure',
]
```

## 📝 调试技巧

### 1. 启用控制台输出

修改 spec 文件：
```python
exe = EXE(
    # ...
    console=True,  # 改为 True 以查看错误信息
    # ...
)
```

### 2. 查看打包日志

```bash
pyinstaller --clean --log-level=DEBUG xrd_app.spec > build.log 2>&1
```

### 3. 测试导入

在打包前测试所有导入：
```python
python check_dependencies.py
```

### 4. 使用 --onefile 模式调试

单文件模式可能暴露不同的问题：
```bash
pyinstaller --onefile main.py
```

### 5. 检查缺失的模块

运行打包后的程序，查看错误信息，然后在 spec 的 `hiddenimports` 中添加缺失的模块。
